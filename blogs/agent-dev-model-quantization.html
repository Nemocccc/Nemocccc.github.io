智能体开发基础知识与模型量化指南

一、大模型量化 — 常见后缀/格式全解

为什么需要量化？

默认模型权重用 BF16/FP16（每个参数 2 字节）。70B 模型 BF16 需要 140GB 显存。量化降低精度来压缩体积：

格式每参数大小相对于 BF1670B 模型显存
FP324 字节2x（更大）280GB
BF16/FP162 字节1x（基准）140GB
FP81 字节节省 50%~70GB
INT81 字节节省 50%~70GB
INT40.5 字节节省 75%~35-40GB

常见后缀逐一解释

精度后缀

后缀全称说明
fp16 / bf16Floating Point 16 / Brain Float 16标准推理精度，无质量损失。BF16 是现代 GPU 原生支持的格式
fp8Floating Point 81 字节浮点，H100/H200 等 Blackwell/Hopper 架构原生支持。质量接近 BF16（保留 ~99%），是 2026 年生产环境的性价比首选
int8Integer 88 位整数，广泛兼容。质量约 BF16 的 97-98%。bitsandbytes 库支持
int4Integer 44 位整数，极致压缩。质量约 BF16 的 93-96%，复杂推理任务下降较明显
nf4Normalized Float 4QLoRA 微调专用的 4 位格式，信息论最优。不用作推理部署

量化方法后缀（重要！）

后缀全称核心原理适用场景
AWQActivation-Aware Weight Quantization分析激活模式，识别重要权重并保护它们，不重要的量化更狠INT4 最佳方案，vLLM 部署首选，质量保留 ~94-96%
GPTQGenerative Pre-Training Quantization逐层量化，利用二阶信息（Hessian 矩阵）最小化误差较早的 INT4 方案，工具支持广泛。质量略低于 AWQ
GGUFGPT-Generated Unified Formatllama.cpp/Ollama 的容器格式。支持混合精度（k-quants），可 CPU+GPU 协同推理本地开发/Ollama 首选，消费级显卡友好
HQQHalf-Quadratic Quantization无需校准数据，解析求解量化问题。几分钟完成 70B 量化快速原型验证，新模型发布当天即可量化
SmoothQuant—平滑激活值异常值，使 INT8 推理更准确数据中心 INT8 部署

GGUF 内部的 k-quant 变体（常见后缀组合）

后缀说明质量 vs BF16
q2_k2 位混合，极致压缩~80-85%（明显降质）
q3_k3 位混合~85-90%
q4_k_m最常用的 4 位平衡点~92-94%
q5_k_m5 位高质量~95-97%
q6_k6 位，接近无损~97-98%
q8_08 位，几乎无损~99%

硬件的选择策略

硬件推荐格式
H100/H200 (Hopper) 或 RTX PRO 6000 (Blackwell) 96GBFP8（质量最高）或 AWQ INT4（70B 单卡）
RTX 5090 32GBAWQ INT4 跑 13-30B，GGUF Q4_K_M
消费级 24GBGGUF Q4_K_M / Q5_K_M 跑 13B

一句话选型口诀

有 H100 用 FP8，显存不够用 AWQ，本地 Mac 用 GGUF，微调用 NF4。

二、智能体 (AI Agent) 开发核心概念

什么是 AI Agent？

AI Agent 是能自主感知环境、做出决策、执行动作的智能系统。不同于单轮问答，Agent 能：

  • 规划：将复杂任务分解为多步
  • 工具调用：使用外部工具（搜索、代码执行、文件操作）
  • 记忆：记住对话历史和中间状态
  • 反思：根据结果自我修正

主流的 Agent 模式

1. ReAct (Reasoning + Acting) 最经典的范式：思考 → 行动 → 观察 → 再思考

Thought: 我需要搜索今天的天气
Action: 调用 search_weather()
Observation: 晴天，25°C
Thought: 天气不错，建议用户出门
Answer: 今天天气很好，适合出去走走

2. Function Calling 模型输出结构化 JSON 调用预定义函数，由框架执行并返回结果。

3. 多 Agent 协作 (Multi-Agent) 多个 Agent 各司其职，通过消息传递协作：

  • Supervisor Agent：分配任务、汇总结果
  • Worker Agent：执行具体子任务
  • Critic Agent：审核/检查质量

4. MCP (Model Context Protocol) 2025-2026 年兴起的新标准协议，让 LLM 和外部工具/数据源通过标准化接口交互。Anthropic 提出，类似 USB 标准——一次接入，到处使用。

Agent 开发常见八股题

Q: Agent 的"记忆"如何实现？ A: 分三层：(1) 短期记忆 = 当前对话上下文窗口 (2) 长期记忆 = 向量数据库存储的嵌入历史 (3) 工作记忆 = 多步推理中的中间状态（通常用 Buffer 或 Graph State 管理）

Q: Tool Calling 和 Function Calling 有什么区别？ A: Function Calling 特指模型输出结构化参数（OpenAI API 格式），Tool Calling 是更宽泛的概念，包括 Function Calling、MCP 工具、自定义 API 等多种形式。

Q: Agent 如何避免无限循环？ A: (1) 最大迭代次数截断 (2) 重复检测（相同工具+相同参数） (3) 带超时的终止条件 (4) 人类介入循环 (Human-in-the-loop)

Q: 什么是 RAG + Agent 的区别？ A: RAG = 检索增强生成，解决知识实时性问题（查资料再回答）。Agent = 能执行动作的自主系统。两者常结合——Agent 用 RAG 作为记忆/知识来源。

Q: 多 Agent 和单 Agent 调度有什么区别？ A: 单 Agent 自己规划+执行+检查。多 Agent 将规划/执行/检查分开，每个 Agent 各司其职，适合复杂任务但增加通信开销。

2026 年 Agent 开发趋势

  • Agentic AI → 生产部署：2026 年 Agent 从演示进入真正生产
  • Multi-Agent 是最热门技能：企业 Agent 需求最大的方向
  • MCP 协议标准化：工具/数据接入从定制化走向标准化
  • Agent 安全成为焦点：Prompt 注入、多步劫持、权限管控
  • Windows Agent Framework：微软在 Build 2026 推出的 OS 级 Agent 平台

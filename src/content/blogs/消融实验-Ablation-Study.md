# 消融实验 (Ablation Study)

## 是什么

控制变量法——把模型/系统的某个组件"切除"掉，观察性能下降了多少，从而判断该组件的贡献度。

名字来源于脑科学：医生切除部分脑区，观察哪些功能丧失，推断该区域的作用。

## 基本原理

```
完整系统 → 性能 95%
去掉组件 A → 性能 80%  → A 贡献了 15%
去掉组件 B → 性能 93%  → B 贡献了 2%（可有可无）
同时去掉 A+B → 性能 70% → A 和 B 有 5% 的交互增益
```

核心逻辑：不依赖直觉，用实验数据证明每个模块是否真的有用。

## 什么时候需要做

### 1. 论文 / 学术汇报
审稿人几乎必问："你怎么证明你加的每个模块都有用？"
消融实验是 ML 论文的标准配置，否则 reviewer 会说"没有消融 = 没有说服力"。

### 2. 迭代新特性 / 新模块时
比如给 RAG 系统加了 reranker + query rewrite，跑一次消融：
- 去掉 reranker → 指标降了 → 保留
- 去掉 query rewrite → 指标没变 → 删掉，少维护一个坑

### 3. 模型压缩 / 剪枝时
移除注意力头、剪掉某些层，看哪些是冗余的，哪些不能动。

### 4. 调试奇怪的现象时
"加了 A 效果反而变差了？" → 单独测 A 的贡献，可能是 A 和已有模块冲突。

## 一个典型例子

做摘要生成系统，三个组件：
- 实体链接（+3% ROUGE）
- 关键词提取（+0.5% ROUGE，可以砍掉）
- 重排序（-1% ROUGE，这个模块负优化）

没有消融，你可能会把所有组件都堆上去，系统变慢变复杂，效果反而没提升。

## 关联概念

- A/B 测试（消融是在固定模型上切组件，A/B 是比较两个不同版本）
- 敏感性分析（消融看组件的贡献度，敏感性看参数变化的影响）
